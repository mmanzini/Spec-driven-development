# CLAUDE.md — SDD Standard

## Project Overview

This is the **Spec-Driven Development (SDD) Standard** — a public, tool-agnostic methodology for structured software development with AI coding agents. It defines how Product Owners, Developers, and AI Agents collaborate through specifications.

All content is Obsidian-flavored Markdown. The standard is designed to be read in Obsidian and published as a reference for community adoption.

## Structure

- **SDD Standard.md** — Hub/index page. Start here.
- **Standard/** — The methodology definition (principles, workflow, roles, glossary).
- **Templates/** — Copyable spec templates for any project adopting SDD.
- **Guides/** — Practical how-to guides for PO/PMs, developers, and teams.
- **Examples/** — Fully worked spec examples.

## Conventions

- **Language:** English.
- **Internal links:** Use Obsidian wiki-link syntax `[[Page Name]]`.
- **Tone:** Professional, prescriptive but not dogmatic. Clear definitions, explicit rationale for each choice.
- **Templates:** Use `> [!info]` callouts for usage instructions, `[BRACKETS]` for user-fillable placeholders, `> [!tip]` for inline guidance.
- **YAML frontmatter:** Templates include `type: template`, `sdd-version`, and `artifact` properties.
- **Versioning:** The standard uses semantic versioning. Current version: 0.1.

## Working with This Project

- The hub page (`SDD Standard.md`) links to all topic files — update both when adding new content.
- File names are descriptive and match the concept they describe.
- Keep files focused — one concept per file, interlinked via wiki-links.
- Do not create files outside this directory structure.
